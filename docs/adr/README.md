# Architectural Decision Records (ADRs)

## Format
Create a new ADR for each architectural decision:

- Filename: `NNNN-short-title.md`
- Keep it short and scannable.
- Include context, decision, alternatives, consequences, and todos.

## Template
```markdown
# ADR NNNN: <Title>

## Status
Proposed | Accepted | Superseded

## Context
<Why this decision is needed>

## Decision
<What we decided>

## Alternatives Considered
- <Option A>
- <Option B>

## Consequences
- <Impact, trade-offs>

## Todos
- <Follow-ups>
```
